# M12 renderer-scale observer probe - 2026-07-21

## Purpose

Test whether the locally Graphics-UI-verified 70-percent Render Scale profile
prevents the FSR renderer failure during an AD 1 observer session.

## Fixed profile

- Windowed resolution: 1920x1080, an installed supported enum value.
- Render Scale: 0.700000, saved by the in-game Graphics UI.
- Upscale Method: Disabled in the observed UI.
- Upscale Quality: Off in the observed UI.

The exact driver profile passed full static validation and a rendered,
enabled-mod 90-second smoke with zero new error-log lines before this probe.

## Autonomous run

1. The driver reached the AD 1 selector, entered Observer Mode, and started
   the live map. See docs/screens/renderer_scale70_probe/observer_start.png.
2. It clicked the observed play control and, after ten seconds, captured a
   running session dated 10:00, 4 January, 1. See
   docs/screens/renderer_scale70_probe/after_play_10s.png.
3. It selected maximum speed and, after another ten seconds, captured 08:00,
   1 June, 1. See docs/screens/renderer_scale70_probe/max_speed_10s.png.
4. The game window disappeared before the next 20-second capture. The
   dedicated server log stopped at version 1.6.1 and the game had written
   G:\antiqvitas_user_data\crashes\Europa Universalis V20260720_235625.

## Result

Fail for observer stability. The exception report records Unhandled Exception
C0000005 in the ffxFsr2ResourceIsNull FSR renderer stack, followed by an NGX
shutdown frame. This is the same failure family as the earlier supported,
full-scale 1920x1080 observer run. The two profiles are materially different
and both menus were smoke-green, so renderer-profile experimentation stops
under the two-attempt recovery rule. The retained 70-percent profile is
accepted only as the menu-smoke checkpoint; it is not evidence for long
observer playback.
