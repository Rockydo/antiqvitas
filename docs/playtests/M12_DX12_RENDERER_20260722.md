# M12 DX12 renderer probe — 22 July 2026

## Scope

Test the only alternate renderer exposed by the installed EU5 Graphics UI after
three Vulkan FSR/NGX crashes prevented the required AD 1-476 Observer run.

## Procedure and evidence

1. Selected the native `DX12` renderer in the installed Graphics screen; the
   setting persisted to the relocated user directory without changing the game
   installation.
2. Confirmed `gfx_dx12_master_context` in the local startup log on the NVIDIA
   GeForce RTX 3080.
3. Loaded a clean all-deferred-market AD 1 Observer start. The driver reached
   18:00 on 13 January, then captured a live DX12 map at 14:00 on 3 March:
   [monitor_0600s.png](../screens/20260722_m12_dx12_background/monitor_0600s.png).
4. Handed playback to an independent maximum-speed observer with hourly
   captures. Its monitor found that the game window had disappeared.

## Result

The game exited at 07:05:19. `G:\antiqvitas_user_data\crashes\Europa
Universalis V20260722_050518\exception.txt` records C0000005 with
`ffxFsr2ResourceIsNull` frames followed by `NVSDK_NGX_D3D12_Shutdown1`.
This is the same failure family as the earlier Vulkan profiles, despite the
different native backend. The clean March checkpoint had no market-manager
self-relation assertion.

The DX12 profile therefore improves the stable interval but cannot provide an
annual, decade, or terminal-date Observer measurement. The M12 campaign gate
remains renderer-blocked pending a material game, GPU-driver, or renderer fix.
